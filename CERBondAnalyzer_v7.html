Total Return Projector — Holding Period Return
Qué hace: proyecta P&L total por bono en un horizonte fijo, descomponiendo Carry (devengo a TIR) + Pull-to-par/Rolldown (revaluación por paso del tiempo + bajada por la curva) + CER accrual (solo bonos CER, indexación del principal) − Funding − Bid/Offer. Ordena los bonos por TR% neto sobre precio. Importante: para bonos CER, la TIR real ya descuenta inflación; para evitar double-count, el default ahora considera TIR real + CER accrual como componente separado.

Z-Spread a Curva NSS (Cashflow-based)
Z-Spread real: spread constante z tal que Σ CF_i / (1 + r_NSS(t_i) + z)^t_i = Precio mercado. Se resuelve por bisección sobre los cashflows reales del bono, descontados con la curva NSS fitted + z. Más riguroso que ΔTNA simple para bonos con diferentes estructuras.
Relative Value Heatmap — Composite Rich/Cheap Score
Score compuesto para lectura rápida de desk: 40% residual vs curva + 25% carry + 20% rolldown + 15% liquidez. Separamos señal, driver dominante y universo para que no quede como un heatmap decorativo.
Implied Forward Inflation Term Structure
Cómo leerlo: el mercado te dice cuánta inflación mensual necesita convalidar cada mes futuro para que sea indiferente comprar CER o LECAP/BONCAP comparable. Barras violeta = TEM forward implícita del mercado (lo que el mercado te está pagando). Línea naranja = tu proyección (grilla de inflación). Puntos de colores = breakeven promedio que cada bono implica. Área sombreada = diferencia acumulada entre tu view y el mercado: si estás arriba del mercado, conviene CER; si estás abajo, conviene tasa fija.
Forward Inflation Matrix — Shock por Mes
Escenario condicional: fijás un mes de inflación y el cuadro te devuelve el sendero requerido para los meses siguientes si querés sostener la curva implícita del mercado. También se muestra el desvío versus la curva BE base para que sea más legible.
Sintético FX — Arbitraje ROFEX vs Tasa Pesos
Compara la tasa implicita devaluatoria del futuro ROFEX contra la tasa en pesos disponible. Si spread > 0 → conviene sintético largo (vender futuro + colocar pesos). Si spread < 0 → sintético corto (comprar futuro + tomar pesos).

Key-Rate Duration (KRD) — Exposición por Bucket
Para qué sirve: rompe la DV01 total de cada bono en buckets de la curva (3M, 6M, 1Y, 2Y, 3Y, 5Y+). A diferencia de la duration agregada, KRD te dice qué parte de la curva mueve a cada bono. Indispensable para cubrir twists (steepener/flattener) sin quedarte sin ver un lado. La suma de las KRDs ≈ DV01 total. $ por 1 bp de shock paralelo en ese bucket solo.
Rotation Planner — Switch Trade Breakeven
Para qué sirve: planificá rotaciones del tipo "vendo A y compro B". Para cada destino B te devuelve el precio breakeven que necesitás pagar por B para que el switch te deje al menos igual (carry, rolldown, CER si aplica) frente a mantener A. Si el ask de mercado está por debajo del BE, el switch te mejora en $ por cada 100VN; si está por arriba, te empeora. También te muestra cuánto cambia la DV01 y el carry mensual del book tras la rotación.
Inflation-Linked Swap Basis — CER vs Synthetic
Para qué sirve: descompone cada bono CER como "LECAP comparable + swap de inflación implícito". Te muestra la TEM fija que recibirías si sinteticás la pata CER con LECAP del mismo plazo, y el basis en bps contra tu proyección. Basis > 0 = el swap implícito te paga por encima de lo que vos esperás que salga la inflación → convalida comprar CER. Basis < 0 = el mercado te paga menos que tu view → convalida tasa fija.
Curva Fitted — Par / Zero / Forward
Para qué sirve: 3 representaciones de la misma curva NSS, cada una útil para algo distinto. Par (yield): TNA al vencimiento que hace que un bono coupón-pagador cotice a la par. Zero (spot): tasa de descuento cero-cupón para cada vencimiento — la que usás para descontar un único flujo. Forward: tasa implícita entre dos fechas futuras (f(t₁,t₂)) — critical para estimar qué espera el mercado para mañana. Cambios entre par y zero indican si la curva tiene mucho cupón; forward inclinado hacia arriba = mercado espera suba de tasas.
Rich / Cheap Residual Z-Score — Histórico
Para qué sirve: no solo mirar el residual vs curva hoy, sino contra su propia historia. Cada vez que entrás a esta pantalla se snapshotea el residual de cada bono en localStorage. Mostramos media, σ y z-score del residual actual vs los últimos N días. |z| > 2 = anomalía de valor (bono rico o barato vs su propio promedio); |z| < 1 = fair. Usalo como filtro primario para no entrar a un spread que está técnicamente barato pero nunca está otra cosa.

📐 Z-Score Residual por Bucket de Duration
Bonos divididos en 4 buckets de duration: 0–0.5, 0.5–1, 1–2, 2+. Para cada bucket fitea curva log y calcula residuo (TNA TF − fit). Z = (residuo − μ_bucket) / σ_bucket. |z|>1.5 = oportunidad relativa dentro del bucket (no contaminada por la curva general). Si hay histórico, usa la media histórica del bono también.

Metodología

API BCRA v4: se mantiene exactamente el fetch del CER desde /estadisticas/v4.0/Monetarias/30. Este archivo sigue siendo HTML+JS ejecutable desde VS Code / navegador.

Ventanas CER: inflación de mes M alimenta CER del 16/M+1 al 15/M+2. Se interpola diario con ancla en día 15.

CER bono: referencia a 10 dias habiles antes de settlement y vencimiento, con feriados BYMA.

LECAP/BONCAP: se agregó un bloque nuevo, separado, sin romper la lógica CER original.

Tasa fija: VF = 100 × (1+TEM)meses 30/360, TEA = (VF/Precio)^(365/dias) − 1, TNA = ((1+TEA)^(dias/365) − 1) × 365/dias.

Duration del scatter: modified duration aproximada para instrumentos bullet/capitalizables: Duration ≈ (dias/365) / (1 + TEA). Para la curva CER se usa la TIR nominal equivalente; para tasa fija, la TEA calculada del título.

Regresión: ajuste logarítmico por mínimos cuadrados: y = a·ln(x) + b, con X = duration y Y = TNA.

Inflacion implicita: se resuelve qué TEM promedio geométrica hace que el CER proyectado iguale al rendimiento del comparable en tasa fija. SOLO se incluyen bonos cuyos meses de inflación relevantes son aún desconocidos. Si el CER de referencia del bono cae antes del 15 del mes (ancla anterior = todo inflación conocida), el breakeven no tiene sentido y el bono se marca como "CER determinado".
